# Proxy.Market → Render (FSA_PROXY_URL)

## 1. Скопировать из личного кабинета

Вкладка **Настройка прокси** → ваша строка:

| Поле | Пример |
|------|--------|
| IP | `109.248.143.43` |
| Порт | `3000` |
| Логин | `hvCmc58a` |
| Пароль | как в кабинете (скопировать точно) |
| Протокол | **HTTP** |

Строка для Render:

```text
http://ЛОГИН:ПАРОЛЬ@109.248.143.43:3000
```

## 2. Render

Dashboard → сервис → **Environment** → Add:

- **Key:** `FSA_PROXY_URL`
- **Value:** строка выше

**Save Changes** → дождаться перезапуска (1–2 мин).

## 3. Proxy.Market — важно

На сайте **ничего настраивать не нужно**, если прокси уже «Новый» и активен.

Если с Render не работает — в кабинете Proxy.Market найдите:

- **Привязка к IP** / **Whitelist** / **Доступ только с моего IP**

Нужно **выключить** или разрешить **любой IP** (достаточно логина и пароля).  
Render ходит с другого адреса, не с вашего Mac.

## 4. Проверка

После деплоя: **Документы → Ozon** — красная подсказка должна исчезнуть.

Или в браузере (после входа): `/api/compliance/fsa-status`

```json
{
  "proxy_configured": true,
  "proxy_reachable": true,
  "reachable": true
}
```

## 5. Частые ошибки

| Симптом | Причина |
|---------|---------|
| «задайте FSA_PROXY_URL» | Переменная не добавлена в Render или сервис не перезапустился |
| «Прокси … недоступен» | Whitelist IP в Proxy.Market или неверный порт |
| «Таймаут ФСА через прокси» | Прокси ок, ФСА временно не отвечает — повторить позже |
