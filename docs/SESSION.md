## Handoff (2026-06-18) — Bin-pack: размечай → упаковывай (шаг 1)

### Проблема (из CARD_LINKING_REVIEW.md)

Pack-шаг финализировал фасовки 1/2/3 и **выкидывал** артикулы из `ai_pool` → сиротские мини-связки. Группировщик `_merge_ai_suggestions_by_use_bucket` работал только над кандидатами ИИ с гейтом `len(cluster) < 2`.

### Сделано

- **`deterministic_bin_pack_from_pool`** — первичная упаковка **всего пула** по `(subjectID, бренд, назначение)` до 30
- **`_facet_aware_pack_bins`** — атомы фасовок целиком, мягкая сортировка по title_base (линейка/вкус — приоритет, не жёсткий split)
- **`ai_pool = pool`** — pack-артикулы **не изымаются** из пула ИИ
- Старый ранний `deterministic_pack_suggestions` в пайплайне заменён на bin-pack
- `_merge_ai_suggestions_by_use_bucket`: убран гейт `< 2`, facet-aware split; ИИ-чанки, уже покрытые bin-pack, пропускаются
- **`default_ai_system_prompt`**: ИИ — дополнение к bin-pack, не дублировать бренд+назначение; мягкий приоритет линейки при добивке

### Следующие шаги (по документу)

1. LLM-разметка «назначения» для категорий вне регэкспа (не-косметика)
2. Упростить промт → разметчик фасетов
3. Метрика: сравнить `uncovered` до/после на реальном каталоге
