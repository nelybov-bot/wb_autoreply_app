# WORKFLOW — обязательный процесс разработки

Этот документ — **единый источник правды** о том, как вести работу над проектом WB AutoReply App (MarketAI).

Применяется ко всем участникам разработки, включая AI-ассистентов в Cursor.

---

## 1. Перед любой задачей (обязательное чтение)

Читать **в указанном порядке** (пропускать только если файл не менялся с прошлой сессии и контекст свежий):

| # | Файл | Зачем |
|---|------|-------|
| 1 | [PROJECT.md](./PROJECT.md) | Что за проект, стек, структура, деплой |
| 2 | [ARCHITECTURE.md](./ARCHITECTURE.md) | Слои, модули, потоки данных |
| 3 | [AI_CONTEXT.md](./AI_CONTEXT.md) | Правила AI, грабли, ключевые файлы |
| 4 | [SESSION.md](./SESSION.md) | Текущее состояние, handoff, открытые вопросы |
| 5 | [TASKS.md](./TASKS.md) | Бэклог и приоритеты |

Дополнительно по необходимости: [BUGS.md](./BUGS.md), [CHANGELOG.md](./CHANGELOG.md), [API.md](./API.md), [ENV.md](./ENV.md).

---

## 2. Перед изменением кода

Кратко зафиксировать (в чате с пользователем):

1. **Что** собираешься изменить и **зачем**
2. **Какие файлы** будут затронуты (список путей)
3. **Риски**: регрессии, WB rate limits, desktop/web, секреты, кэш статики, БД

Не начинать правки кода, пока это не сформулировано (кроме тривиальных однострочных правок по явному запросу).

---

## 3. После любого изменения кода

Обновить документацию **в той же сессии**, до завершения задачи:

| Условие | Файл |
|---------|------|
| Всегда | [SESSION.md](./SESSION.md) |
| Всегда | [CHANGELOG.md](./CHANGELOG.md) |
| Изменилась архитектура, API, модули | [ARCHITECTURE.md](./ARCHITECTURE.md) |
| Новые договорённости, грабли, правила | [AI_CONTEXT.md](./AI_CONTEXT.md) |
| Найден или исправлен баг | [BUGS.md](./BUGS.md) |
| Новая или завершённая задача | [TASKS.md](./TASKS.md) |

Также при изменении `app.js` / `styles.css` — увеличить `?v=` в `index.html` (см. AI_CONTEXT.md).

---

## 4. Завершение задачи (handoff)

В [SESSION.md](./SESSION.md) добавить блок **Handoff**:

- **Сделано:** что изменено, commit (если есть)
- **Осталось:** незакрытые пункты, риски
- **Следующий шаг:** одна конкретная рекомендация

---

## 5. Длинный диалог или смена задачи

Если контекст переполнен или задача **существенно** меняется:

1. Предложить пользователю **закончить текущую сессию**
2. **Полностью обновить** SESSION.md (handoff + актуальный статус)
3. Убедиться, что TASKS.md и BUGS.md отражают текущее состояние
4. В новом чате снова выполнить шаг 1 (чтение docs)
